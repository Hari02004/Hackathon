9. EVENT REGISTRATION FLOW
================================================================================

1. User Views Events
    ↓
2. Clicks "Explore More" on Event
    ↓
3. Event Details Modal Opens
    ↓
4. Clicks "Register Now" Button
    ↓
5. Registration Modal Opens
    ├── Auto-filled Fields
    │   ├── Name (from profile)
    │   ├── Email (from profile)
    │   ├── Phone (editable)
    │   └── Department (from profile)
    │
    └── Editable Fields
        ├── Year (dropdown)
        ├── Semester (text)
        └── Admission # (text)
    ↓
6. User Fills/Edits Details
    ↓
7. Clicks "Register Now" Button
    ├── Validation
    │   ├── Name required ✓
    │   ├── Email required ✓
    │   └── Phone required ✓
    │
    └── Check if Already Registered
    ↓
8. API Call to Backend
    POST /api/event-registrations/register
    ├── Body: eventId, name, email, phone, etc.
    └── Response: success/error
    ↓
9. Backend Processing
    ├── Check Event Exists
    ├── Check Duplicate Registration
    ├── Create Registration Record
    ├── Send Confirmation Email
    └── Return Success
    ↓
10. Frontend Success
    ├── Show "✅ Successfully Registered"
    ├── Add to User Registrations
    ├── Disable "Register Now" Button
    ├── Show "✅ Registered" Button
    └── Auto-close Modal after 2.5s
    ↓
11. User Can View Registration
    ├── In Profile Page
    ├── In My Registrations Section
    └── Status: Pending/Attended
